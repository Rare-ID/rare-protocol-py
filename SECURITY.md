# Security Policy

## Reporting

Please report security issues privately to `security@rareid.cc`.

Include:

- affected package and version
- impact summary
- reproduction steps or proof of concept
- whether the issue affects protocol rules, implementation, or documentation

Do not open public GitHub issues for undisclosed vulnerabilities.

## Scope

Security reports are especially useful for:

- signature verification bypasses
- replay protection failures
- token validation gaps
- key handling mistakes
- trust-boundary confusion that could cause unsafe integration defaults

## Response

We will acknowledge valid reports, investigate severity, and coordinate a fix and disclosure timeline before public release when appropriate.
